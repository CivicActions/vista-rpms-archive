# Quickstart: Document Extraction Pipeline

**Feature**: 001-doc-extraction  
**Date**: 2026-01-17

## Prerequisites

- Python 3.11+
- GCS credentials configured (GOOGLE_APPLICATION_CREDENTIALS or gcloud auth)
- Source bucket with `index.json` at root (generated by `index_files.py`)

## Installation

```bash
# Install uv if not already installed
curl -LsSf https://astral.sh/uv/install.sh | sh

# Create virtual environment and install dependencies
uv venv
source .venv/bin/activate
uv pip install -e .
```

## Configuration

Create `config.toml`:

```toml
[gcs]
source_bucket = "my-archive-bucket"
source_prefix = "data/"
cache_bucket = "my-archive-bucket"
cache_prefix = "cache/"

[processing]
workers = 8
max_pending = 20
timeout = 120

[extraction]
max_pages = 500
max_file_size = 52428800  # 50MB
```

## Usage

### Basic Extraction

```bash
# Run extraction with config file
python extract.py --config config.toml

# Override workers via CLI
python extract.py --config config.toml --workers 4

# Dry run (show what would be processed)
python extract.py --config config.toml --dry-run
```

## Environment Variables

| Variable | Description | Default |
|----------|-------------|---------|
| GOOGLE_APPLICATION_CREDENTIALS | Path to service account JSON | (gcloud default) |
| EXTRACTION_LOG_LEVEL | Logging level | INFO |
| EXTRACTION_WORKERS | Override worker count | os.cpu_count() |

## Common Tasks

### Re-process Failed Files

```bash
# Extract file paths from error log and reprocess
grep "FAILED" extraction_errors.log | cut -d'|' -f3 | xargs python extract.py --files
```

### Check Cache Coverage

```bash
# Show cache statistics
python extract.py --config config.toml --stats-only
```

### Process Specific Directory

```bash
# Only process files under a specific prefix
python extract.py --config config.toml --prefix data/reports/2025/
```

## Troubleshooting

### "Permission denied" errors
- Verify GCS credentials: `gcloud auth application-default print-access-token`
- Check bucket IAM permissions (Storage Object Viewer for source, Storage Object Creator for cache)

### Out of memory
- Reduce `--workers` count
- Reduce `max_pending` in config
- Check for very large files exceeding `max_file_size`

### Slow processing
- Increase `--workers` (up to 2x CPU count for I/O-bound work)
- Check network bandwidth to GCS
- Consider running on a VM in the same region as the bucket
